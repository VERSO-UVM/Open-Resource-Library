# Where to Start

Not sure where to begin? This page helps you figure out your starting point and points you to the right guides.

## Are You Brand New to Programming?

If you've never written code before and don't have any developer tools installed, follow this path in order:

1. **[Set Up Your Environment](./environment-setup)** — Install the tools you need: a code editor, Git, Python, and SSH keys for GitHub. Do this first, before anything else.
2. **[Learn Markdown](./markdown-basics)** — Markdown is the simple formatting language used in this library, on GitHub, and in most documentation. It takes about 15 minutes to learn.
3. **[Command Line](/concepts/command_line)** — Learn how to navigate your computer using the terminal. Most developer tools are run from the command line.
4. **[Git](/tools/git)** — Learn how to track changes to your code using Git.
5. **[GitHub](/tools/github)** — Learn how to share code and collaborate using GitHub.
6. **[Python](/tools/python)** — Learn the basics of Python, the most common language in research and data science.
7. **[Make Your First Contribution](./your-first-contribution)** — Put it all together by contributing to a real open source project.

## Already Have Some Experience?

If you have coding experience but are new to open source or research computing, you can skip the environment setup and jump straight to:

- **[Git](/tools/git)** and **[GitHub](/tools/github)** if you haven't used version control before
- **[Testing](/concepts/testing)** and **[Debugging](/concepts/debugging)** for software quality fundamentals
- **[Security Basics](/concepts/security_basics)** — especially important for research computing
- **[APIs](/concepts/apis)** for fetching and working with external data
- **[CI/CD](/concepts/ci_cd)** for automating testing and deployment

## What You'll Need

To work through the beginner path, you'll need:

- A computer running **macOS, Windows, or Linux**
- An internet connection
- A free [GitHub account](https://github.com) (you can create one during the environment setup)
- About 1–2 hours for the initial setup

You don't need to buy anything. All the tools in this library are free and open source.

## A Note on the Terminal

Many of the guides in this library use the **terminal** (also called the command line, shell, or console). If you've never used it before, it can look intimidating — but it's just a text-based way to interact with your computer.

You'll learn everything you need in the [Command Line](/concepts/command_line) guide. For now, just know that when you see a code block that starts with `$`, the `$` is not part of the command — it just indicates that the command is run in a terminal. For example:

```bash
$ git --version
```

means: open your terminal and type `git --version`, then press Enter.
